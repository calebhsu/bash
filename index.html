Dear Bash,

Though best learned on a Linux-based machine,

I've come to appreciate how cool I look typing things into a terminal (even on Windows. Maybe especially on Windows).
Thanks for making navigation, file creation, and file removal so simple.
I don't even have to touch my mouse.

In order to do anything remotely useful,

we have to know how to navigate. Correct me if I'm wrong, ok?
First off, I list the contents of my present directory using ls.
Then in order to change directories, I cd directoryname. Again,
I can now ls to see the contents of my new directory.

Some time-saving shortcuts:

  • pwd: displays the present working directory (in case I ever get lost)
  • cd ~: change to home directory
  • cd /: change to root directory

To cd multiple levels, simply duplicate the path name:
cd directory/nextlevel
Or if moving outwards, replace each name in the path with ..
cd ../.
Did I get that right?

Onto something potentially dangerous...

...creating and removing directories and files. Really creation is the simple part, it's
removal that could potentially get sticky.
Make a new directory using mkdir directoryname. I know now I can ls to make sure it's there.
But what if I want the power to delete stuff? Well. I love and hate how hard you've made it to delete, Bash.
Remove a directory using rmdir directoryname. And it'll be gone, no warning or anything...if it's already an empty directory. But if it's full,
I have to go in and delete the contents. Which, granted I can do with some fancy flags but...
...I can never remember them. It's ok, I'm content doing that throught the UI anyways.

How's about files? Also simple. Just watch what you're removing.
Create a file using touch filename. Don't forget the extension ending!
Remove a file using rm filename.

Renaming can be a little confusing due to naming conventions--
just remember that mv does not stand for "move directory". Instead,
rename files using mv oldname newname.
Gloriously few clicks. Like I said, thanks Bash!

In the name of shell scripting,


Caleb
